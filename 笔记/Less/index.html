CSS预处理器Less

三种预处理器

Less、Sass、Stylus

作用：方便开发者编写代码，结构更清晰，更简洁，提高编码效率

一.less简介

less是一种动态样式语言，增加了变量，函数，mixin等特性，既可以在客户端运行，也可以借助node.js在服务器端运行。

官网：http://lesscss.cn/

文件后缀为.less，需要编译成css文件方可使用

二.编译方式：

1.通过外部编译器，如考拉koala

2.通过命令行，需要安装node.min.js运行环境

3.通过引入less.js脚本文件在线编译（style中type=”text/less”，脚本一般在底部引入)，性能较差

三.注释

以//开头的注释不会编译到css文件中

以/**/包裹的注释会编译到css文件中

四.变量

使用@声明变量 @变量名:含义 ，根据变量不同的含义有不同的引用方式：

1.作为普通属性值：直接@color

2.作为选择器或属性名：#@{selector}

3.作为URL：@{url}

变量具有延时加载的特点，在加载完作用域（块级）的代码后再赋值给变量

五.嵌套规则

1.基本嵌套规则 – 嵌套的双方为HTML的父子关系，编译后的css中两者之间加上空格

2.&的使用 – 在选择器前加上&表示同级

六.混合mixin

混合就是将一系列属性从一个规则集引入到另一个规则集的方式（相当于ctrl+c和ctrl+v)

使用. 的形式 定义,一般会放在额外的文件夹中作为混合库（API）使用，通过 @import “文件名” 引入。

目的:提高代码的复用率

1.普通混合–混合会编译到css文件中

.center{

margin:auto;

}

2.不带输出混合–混合不会编译到css文件中

.center(){

margin:auto;

}

3.带参数混合

.center(@c){

margin:auto;

background: @c;

}

4.带参数有默认值混合

.center(@c:pink){

margin:auto;

background: @c;

}

5.多个参数混合

.center(@w:100px,@c:pink){

margin:auto;

background: @c;

width: @w;

}

6.命名参数

在引用混合时实参指定变量名

#div{

.center(@c:pink);

}

7.匹配模式

额外定义同名混合，可在括号内插入一个标识符代表某模式，在该混合内部编写其特有的的代码，原混合的第一个参数必须为@_且形参数量与匹配模式的相同

8.arguments变量

实参列表，通过@arguments引用，简写代码

.border(@1,@2,@3){

border:@arguments;

}

七.计算

加减乘除，不同于css的cacl方法，只需计算的一方带有单位即可

八.继承

性能比混合高，灵活性较低，与类相似不带参数

：extend(.类名) 表示继承

多个选择器包含公共代码

九.避免编译

保持代码的原样编译到css中
